# Verbose Output

As you debug and build your AI agent, you'll probably want to dump a lot more context into the console... but at the same time, we don't want to make the user experience of our CLI tool too noisy...

Let's add an optional command-line flag, `--verbose`, that will allow us to toggle "verbose" output on and off. When we want to see more info, we'll just turn that bad boy on.

## Assignment

1. [x] Add a new optional command-line argument, `--verbose`. `argparse` makes this easy – `"store_true"` means the argument will be parsed as a boolean value of `True` if the flag is set, otherwise `False`. The optional argument can appear either before or after the prompt, and its variable will be accessible as `args.verbose` (assuming you named your parsed arguments `args`).
   
    ```python
    parser.add_argument("--verbose", action="store_true", help="Enable verbose output")
    ```

2. [x] If the --verbose flag is included, the console output should include:
    - [x] The user's prompt: "User prompt: {user_prompt}"
    - [x] The number of prompt tokens: "Prompt tokens: {prompt_tokens}"
    - [x] The number of response tokens: "Response tokens: {response_tokens}"
  
Otherwise, this metadata should *not* be printed – only the response.

1. [x] Try running your script with and without the `--verbose` flag to verify the difference in output, e.g.,

    ```python
    uv run main.py "What is the meaning of life?" --verbose
    ```

    ```python
    uv run main.py "How much wood would a woodchuck chuck if a woodchuck could chuck wood?"
    ```
